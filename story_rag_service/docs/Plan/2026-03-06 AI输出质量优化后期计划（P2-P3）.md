## P2：RAG 检索质量提升

### 2026-03-06 混合检索 + Cross-Encoder 重排

**状态**：✅ 已完成（2026-03-06）

**目标**：将当前"规则匹配 + 向量检索简单合并"升级为分数归一化的混合检索管道，命中精度预计提升 15–20%。

**涉及模块**：
- `story_rag_service/application/story_generation/retrieval.py`
- `story_rag_service/services/vector_store/manager.py`

**方案概要**：

1. **BM25 关键词层**（新增）  
   - 在 `retrieve_rag_context` 中，parallel 运行 BM25（`rank-bm25` 包）对所有 lorebook 条目做关键词得分计算。  
   - 仅在内存中索引当前世界的条目（每次 world_id 变化时重建，开销极小）。

2. **RRF 融合**（新增）  
   - 将向量得分与 BM25 得分用 Reciprocal Rank Fusion（`score = 1/(k+rank)`，k=60）归一化合并。  
   - 替换现有的 list 拼接去重逻辑。

3. **Cross-Encoder 精排**（可选，离线模式）  
   - 若 `ENABLE_RERANKER=true`（config.py 新增开关），则对 top-10 候选加载  
     `cross-encoder/ms-marco-MiniLM-L-6-v2`（<100MB，可离线缓存到 `data/huggingface_cache/`）进行精排，再取 top-k。  
   - CPU 推理时间 < 50ms（10条候选），不影响端到端延迟体感。

**依赖变动**：
- `requirements.txt` 新增：`rank-bm25>=0.2`（可选：`sentence-transformers>=2.5` 用于重排）
- `config.py` 新增：`enable_reranker: bool = False`

**前端/DB**：无改动。

### 2026-03-06 检索结果压缩（长 Lorebook 条目摘要化）

**状态**：✅ 已完成（2026-03-06）

**目标**：超过 300 字的 lorebook 条目命中后，先经 LLM 摘要到 80 字再注入 prompt，节省约 15–30% system prompt token 消耗。

**涉及模块**：
- `story_rag_service/application/story_generation/retrieval.py`（压缩触发）
- `story_rag_service/services/vector_store/manager.py`（条目长度判断）

**方案概要**：
- 在 `_select_vector_contexts` 返回后，对 `content` 长度 > 300 字的条目，用简短 prompt 调用同一 LLM 实例（fire-and-forget 或 await），缓存到 `data/chroma_db/` 附加的 SQLite 表 `lorebook_summaries(entry_id, summary, created_at)`。
- 缓存命中直接返回，避免重复 LLM 调用。

**前端/DB**：新增 `lorebook_summaries` SQLite 表（自动建表，无 migration 风险）。

## P2：Prompt 模块化工程化重构

### 2026-03-06 PromptComponent 架构重构

**状态**：✅ 已完成（2026-03-06）

**目标**：将 `prompt_builder.py` 中 300 行的 f-string 拼接重构为可独立测试、独立禁用的 `PromptComponent` 架构，为后续 A/B 测试、prompt 版本化打基础。

**涉及模块**：
- `story_rag_service/services/story_generation/prompt_builder.py`（核心重构）
- 对外导出函数 `build_system_prompt` 签名**不变**，内部实现改造

**方案概要**：

```python
class PromptComponent:
    name: str
    priority: int          # 裁剪优先级（越低越先被裁剪）
    required: bool = False # True 则不可裁剪
    enabled: bool = True
    
    def render(self, **context) -> str: ...
    def estimate_tokens(self) -> int: ...
```

各组件独立类：`WorldContextComponent`、`HistoryContextComponent`、`SummaryMemoryComponent`、`StyleComponent`、`AtmosphereComponent`、`RoleplayComponent`、`CoreInstructionComponent`。

`build_system_prompt` 改为：组装 components → 排序 → 调用 `_trim_to_budget` → `join`。

**前端/DB**：无改动。

## P3：前端用户体验增强

### 2026-03-06 输入建议 + 指令模板库

**状态**：✅ 已完成（2026-03-06）

**目标**：在故事输入框区域提供情境感知的快捷行动建议，降低用户输入门槛，引导高质量 prompt。

**涉及模块（前端）**：
- `frontend/src/views/StoryView.vue`（主界面）
- `frontend/src/stores/ragStory.ts`（将 `choices` 字段从服务端响应中解析并存储）
- `frontend/src/components/` 新增 `StoryInputSuggestions.vue`

**涉及模块（后端）**：
- `story_rag_service/api/v2/schemas.py`：`V2GenerateResponse.choices` 字段已存在，无需改动。
- `story_rag_service/api/v2/story/generation_routes.py`：流式端点补充 `choices` 解析（当 `mode=choices` 时）。

**方案概要**：
- 默认 `mode=choices`（可由用户设置关闭），AI 响应末尾自带 `[A][B][C]` 选项。  
- 前端解析 `choices` 数组，渲染为可点击的快捷按钮，点击自动填充输入框。  
- 新增"指令模板"下拉菜单，包含：氛围切换、时间跳跃、视角切换、强制剧情走向等 6–8 个预设。

**前端改动**：`StoryView.vue`、`ragStory.ts`、新增 `StoryInputSuggestions.vue`  
**后端改动**：`generation_routes.py` 流式路径补 choices 解析  
**DB**：无
