# Plan_2026-04-10_故事记忆系统合并实施任务清单

## 1. Planning Summary

### 1.1 目标

将“故事记忆系统合并重构方案”拆成可执行实施任务，并按阶段记录完成情况。实施顺序遵循：

1. 先统一契约
2. 再统一前端记录模型
3. 再把聚合读模型切到主路径
4. 最后降级旧实体状态能力并清理兼容层

### 1.2 约束

1. 必须遵循单一职责原则，避免把更多逻辑继续堆进 `story_generator.py` 或 `storySession.ts`
2. 涉及故事主链路的改动必须联动前后端契约
3. 每完成一个阶段，都要在本清单内记录状态与完成内容
4. 旧接口与旧字段在迁移完成前必须保持兼容
