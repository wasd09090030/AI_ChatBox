# 2026-03-09 世界书收口并下线独立角色卡

## 本次改动目标

这一批改动把角色管理继续收口到 Lorebook：

1. 删除前端独立角色卡管理页。
2. 删除后端独立 character-card API。
3. 明确旧角色卡数据全部废弃，不做迁移。
4. 在世界书页面中新增“关键角色”独立区域，并把主角人设管理一起并入，避免功能回退。

## 前端改动

### 删除独立角色卡管理页

已移除：

1. [frontend/src/router/index.ts](frontend/src/router/index.ts) 中的 `dashboard-characters` 路由。
2. [frontend/src/components/layout/AppSidebar.vue](frontend/src/components/layout/AppSidebar.vue) 中的“角色卡”导航项。
3. [frontend/src/views/DashboardCharactersView.vue](frontend/src/views/DashboardCharactersView.vue)。
4. [frontend/src/views/RoleplayView.vue](frontend/src/views/RoleplayView.vue) 旧独立角色对话页残留文件。
5. [frontend/src/components/characters/CharacterCardsTab.vue](frontend/src/components/characters/CharacterCardsTab.vue) 旧角色卡列表组件。

### 清理旧角色卡前端数据链路

已移除：

1. [frontend/src/services/roleplayService.ts](frontend/src/services/roleplayService.ts) 中的 character-card 类型与 CRUD API。
2. [frontend/src/domains/roleplay/queries/useRoleplayQueries.ts](frontend/src/domains/roleplay/queries/useRoleplayQueries.ts) 中的角色卡 query/mutation。
3. [frontend/src/domains/roleplay/api/roleplayApi.ts](frontend/src/domains/roleplay/api/roleplayApi.ts) 与 [frontend/src/domains/roleplay/queries/useRoleplayCatalogQueries.ts](frontend/src/domains/roleplay/queries/useRoleplayCatalogQueries.ts) 中的旧角色卡目录查询。

### 世界书页面重构

核心落点：

- [frontend/src/views/DashboardLorebookView.vue](frontend/src/views/DashboardLorebookView.vue)
- [frontend/src/components/lorebook/EntrySheet.vue](frontend/src/components/lorebook/EntrySheet.vue)

现在世界书页包含这些层次：

1. `关键角色` 标签：只展示 `role_tier = principal` 的角色。
2. `角色` 标签：只展示普通 NPC。
3. `地点` / `事件` 标签：保持原逻辑。
4. `主角人设` 标签：承接原 DashboardCharacters 页里的人设管理，改为在世界书页统一维护。

同时，创建角色时支持预设：

1. “关键角色”按钮会直接以 `principal + dialogue_enabled=true` 打开角色表单。
2. “角色”按钮会以普通 NPC 预设打开角色表单。

## 后端改动

### 删除独立 character-card API

已修改：

- [story_rag_service/api/v2/roleplay_routes.py](story_rag_service/api/v2/roleplay_routes.py)

删除了以下接口：

1. `GET /api/v2/roleplay/character-cards`
2. `POST /api/v2/roleplay/character-cards`
3. `GET /api/v2/roleplay/character-cards/{card_id}`
4. `PUT /api/v2/roleplay/character-cards/{card_id}`
5. `DELETE /api/v2/roleplay/character-cards/{card_id}`

保留：

1. persona 相关 API
2. story_state 相关 API

### 移除运行时对旧角色卡的依赖

已修改：

- [story_rag_service/services/story_generation/context_helpers.py](story_rag_service/services/story_generation/context_helpers.py)
- [story_rag_service/api/v2/story/session_routes.py](story_rag_service/api/v2/story/session_routes.py)

变化：

2. 创建故事 session 时，不再尝试从旧角色卡注入 `first_message`。

3. [story_rag_service/services/roleplay_profiles/mappers.py](story_rag_service/services/roleplay_profiles/mappers.py) 删除 character-card 行映射。

5. [story_rag_service/scripts/seed_character_cards.py](story_rag_service/scripts/seed_character_cards.py) 删除。

## 当前结果

现在系统中的角色职责已经更清晰：

1. 世界内角色统一放入 Lorebook 管理。
2. 关键角色是 Lorebook 角色的强化形态，不再依赖旧角色卡。
3. 主角人设仍保留，但管理入口已并入世界书页面。
4. 独立角色卡页面和后端角色卡 API 都已下线。

## 仍需后续处理的部分

1. API 参考文档中仍有旧 character-card 接口描述，需要在下一轮清理。
2. `character_card_id` 字段仍在部分 story/session schema 中保留兼容位，虽已不参与主流程，可在后续进一步删除。
3. Roleplay SQLite 基础设施里仍保留旧表初始化语句，这不影响运行，但属于待清理遗留项。
