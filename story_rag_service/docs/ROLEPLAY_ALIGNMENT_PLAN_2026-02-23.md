# RP 能力对齐执行清单（平衡版，8-12 周）

更新时间：2026-02-23

## 1. 目标与范围

目标：在现有 v2 生成链路基础上，升级到“世界书驱动 + 角色卡 + 主角人设 + 可控剧情推进”的单角色沉浸式 RP 体验。

范围（本期）：
- 单角色沉浸 RP（不做多角色群聊调度）。
- 可自定义：世界书、角色卡、主角人设、剧情结构。
- 兼容现有接口：`POST /api/v2/story/generate` 保持可用。

非目标（本期不做）：
- 完整复刻 SillyTavern 扩展生态。
- 大规模前端重设计。
- 复杂脚本引擎（如 STScript 等价物）。

## 2. 能力拆分与验收口径

### A. 角色卡一等实体（CharacterCard）

交付：
- 新增角色卡数据模型（基础字段 + 高级字段）。
- 支持角色卡 CRUD 与会话绑定。

验收标准：
- 同一会话绑定角色卡后，连续 10 轮输出语气、称谓、行为倾向一致。
- 未绑定角色卡时行为退化为当前默认逻辑，不报错。

### B. 主角人设（Persona）

交付：
- 新增主角人设模型，支持会话级绑定。
- 在提示词装配中注入 persona 段。

验收标准：
- 切换 persona 后，新生成内容立即体现人设差异。
- 历史消息不被破坏，行为为“新轮次生效”。

### C. 世界书激活引擎（Rule + Vector Hybrid）

交付：
- 在向量检索之外新增可控激活规则：关键词/正则、优先级、预算、互斥组。
- 生成响应返回命中条目日志（可观测）。

验收标准：
- 指定关键词可稳定触发对应条目。
- 预算超限时按优先级裁剪，结果可解释。

### D. 剧情状态机（StoryState）

交付：
- 会话维护章节、目标、冲突、线索、关系弧等状态。
- 每轮先更新状态，再生成文本。

验收标准：
- 30 轮长对话中，剧情主线不漂移，目标状态可追踪。
- 关键状态变更有日志可回放。

### E. 记忆体系升级

交付：
- 短期窗口 + 向量长记忆 + 摘要记忆三层。
- 摘要记忆按阈值更新。

验收标准：
- 上下文 token 使用下降且生成稳定性提升。
- 长会话中关键事实召回率提升。

## 5. 风险与回滚

风险：
1. 提示词注入段过多导致 token 超预算。
2. 角色卡/世界书冲突导致输出不稳定。
3. 多源记忆冲突（短期/摘要/向量）导致事实漂移。

缓解：
- 引入预算分配器（硬阈值 + 裁剪优先级）。
- 建立注入顺序规范与冲突裁决规则。
- 摘要记忆设定“可信字段优先级”。

回滚：
- 通过特性开关关闭 RP 增强段，回退到当前生成链路。
- 保持协议向后兼容，不移除现有字段。

当前可用开关（`story_rag_service/config.py`）：
- `RP_STORY_STATE_ENABLED=false`：关闭剧情状态更新节点。
- `RP_SUMMARY_MEMORY_ENABLED=false`：关闭摘要记忆注入与更新。
- `RP_DEBUG_PAYLOAD_ENABLED=false`：关闭响应中的调试负载（`activation_logs`、`story_state_snapshot`、`summary_memory_snapshot`）。
